Хэш-функции

Хэш-функция - математическое преобразование, в результате которого для исходных данных произвольной длины формируется последовательность фиксированной длины, при этом по полученной последовательности практически невозможно восстановить исходное сообщение. Любое изменение исходных данных влечет за собой значительное изменение последовательности.

Или более простыми словами: мы получаем уникальное значение для любого набора данных (пароль, текст, файл)

MD5 (Message Digest 5) — 128-битный алгоритм криптографического хеширования и обычно это последовательность из 32 шестнадцатеричных символов. Чаще всего используется для хеширования логинов и паролей.

SHA-1 (Secure Hash Algorithm 1) — 160-битный алгоритм криптографического хеширования. Выглядит как последовательность из 40 шестнадцатеричных символов. Используются в системах контроля версий системах электронной подписи, хеширования логинов и паролей, а также для построения кодов аутентификации.

SHA-2 (Secure Hash Algorithm Version 2) — семейство криптографических алгоритмов — однонаправленных хеш-функций, включающее в себя алгоритмы SHA-224, SHA-256, SHA-384 и SHA-512. Используется для хеширования логинов, паролей, так же для создания электронной цифровой подписи сертификата.

Как показали исследования, алгоритмы SHA-2 работают в 2—3 раза медленнее других популярных хеш-алгоритмов MD5, SHA-1, Tiger и RIPEMD-160.

Для усложнения “взлома” хэша часто применяют следующую модификацию, которая называется “соль”, или термин “подсоленый хэш”. Но об этом расскажем когда будем обсуждать популярные криптографические алгоритмы.
